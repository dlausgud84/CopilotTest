## 개요
이 문서는 본 프로젝트의 백엔드/프론트엔드 공통 개발 패턴을 빠르게 참고하기 위한 실무 가이드입니다.

## 백엔드 작성 패턴
- 패키지: `com.dit.<domain>.<layer>`
- 계층 역할 분리: Controller(입출력), Service(비즈니스), Mapper(DB)
- MyBatis XML 매핑 사용: `resources/mappers/**/*.xml`
- 공통 예외/응답은 `backend/modules/common` 재사용

## 프론트엔드 작성 패턴
- Nuxt 페이지는 `pages/`, 공통 UI는 `components/`, 재사용 로직은 `composables/`
- composable 명은 `use*` 접두어 사용
- API 호출은 composable에서 처리하고 화면 컴포넌트는 표현/상태 위주로 유지

## 테스트 및 검증
- 백엔드: 단위/통합 테스트, 경계값/예외 케이스 포함
- 프론트엔드: 컴포넌트 렌더링, 상태 전이, API 실패 케이스 검증
- 변경 후 문서 인덱스 및 링크 유효성 점검 필수
